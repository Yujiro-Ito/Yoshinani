# よしなに (Yoshinani) — プロジェクト規約（Claude 用）

## 【必須】コミット前のコードレビュー

**コード変更を含む `git commit` の前には、必ず先にコードレビューを実行すること。**

- レビューは **feature-dev のコードレビュー機能**（`feature-dev:code-reviewer` サブエージェント、
  または `/code-review`）で行う。直前のレビュー対象は「今からコミットする差分」。
- 重大度の高い指摘（バグ・正確性・セキュリティ・規約違反）は**修正してからコミット**する。
  修正しない場合は理由を応答／コミットメッセージに明記する。
- 対象は**実コード**（C++ / CMake / PowerShell 等）の変更。
  ドキュメント・仕様（`.md`）のみの変更はレビュー不要。
- 標準フロー: **変更 → コードレビュー → 指摘対応 → ビルド & ctest 緑 → コミット**。

> hook では LLM レビューを実行できないため、この規約は「Claude が必ず守る指示」として運用する。

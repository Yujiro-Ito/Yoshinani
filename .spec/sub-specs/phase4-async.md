# 4-A async 化（B 方式）

## 概要

§6.5 の継ぎ目（①request_id付き非同期IF ②変換待ちリスト ③投入順キュー）を実働させ、
**変換を待たない**ようにする。B方式＝**未確定のまま保持**し、変換完了したものから**投入順に確定**。

## スコープ

- `ConversionQueue` の `MAX=1` を外す（複数の変換待ちを保持）。
- 変換中も preedit（IME所有）のまま保持し、領域を他アプリに触らせない（MAIN_SPEC §6.5 の B 表）。
- `IKanaKanjiConverter.Convert` を「投げて待たない」に変更（callback で後から確定）。

## 前提

- Phase 3 完了（同期で 1 件動作）。3-A/3-D の「形」が入っていること。
